# IPC 란?

IPC 란 Interprocess Communication 의 약자이며 process 간 협력 메커니즘을 의미합니다. 크게 두가지 방법으로 나눌 수 있습니다.

1. Message Passing: kernel 을 통해 message 를 전달하는 방법
2. Shared Memory: 서로 다른 process 간 일부 메모리를 공유하게 하는 방법


## Message Passing

<img src="/assets/images/os/ipc_message_passing.png" width="400" height="500">

두 프로세스 사이에 공유 변수를 사용하지 않고 통신하는 시스템 입니다. 무조건 kernel 을 통해 message 를 전달하는 방식 입니다. 
message passing 방법 또한 두가지 유형으로 나눌 수 있습니다.

### Direct Communication

<img src="/assets/images/os/ipc_direct_communication.png" width="600" height="200">

통신하려는 process 의 이름을 명시적으로 표시하며 사용하는 방법입니다.

<img src="/assets/images/os/ipc_indirect_communication.png" width="600" height="200">

`mail box` 또는 `port` 를 통해 message 를 간접 전달하는 방식입니다. 물론 mail box 도 kernel 에 존재합니다. mail box 에 들어간 message는 경우에 따라 아무 process 가 해당 message 를 볼 수 있습니다.

## Shared Memory

<img src="/assets/images/os/ipc_shared_memory.png" width="400" height="500">

원칙적으로는 process 들은 독립적인 주소공간을 가지고 있어 자신의 code, data, stack 영역만 볼 수 있습니다. 하지만 shared memory 방식은 일부 주소공간을 두 메모리가 공유하는 방식입니다.
